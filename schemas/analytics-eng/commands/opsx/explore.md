Enter explore mode - a thinking partner for data exploration.

**Input**: The argument after `/opsx:explore` can be a question, discrepancy, table name, analysis name, or nothing.

---

Invoke the `openspec-explore` skill using the Skill tool, then follow its stance.

The skill establishes explore mode as:
- A thinking stance, not a workflow
- For investigation and understanding, not execution
- Data-grounded with BigQuery, DBT, Looker queries
- Visual with ASCII diagrams for lineage and comparisons

**Remember**: Explore mode is for thinking. To execute an analysis, exit and run `/opsx:analyze`.
